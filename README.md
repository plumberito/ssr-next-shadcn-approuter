# ssr-next-shadcn-approuter

Template for a Server-Side Rendered (SSR/SSG) application using Next.js 15 App Router, Shadcn/ui, and Tailwind CSS v4. Intended to be used as a starting point — another LLM or developer can scaffold this template and have a production-ready structure from which end users can begin building immediately.

---

## Stack

| Layer | Technology |
|---|---|
| Framework | Next.js 15 (App Router) |
| Build tool | Turbopack (via `next dev --turbopack`) |
| Language | TypeScript |
| Router | App Router — file-based, nested layouts, RSC-first |
| UI components | Shadcn/ui |
| Styling | Tailwind CSS v4 (via PostCSS) |
| Package manager | pnpm |

## Key configuration files

### `postcss.config.mjs`

Tailwind v4 uses PostCSS instead of a `tailwind.config.js`:

```js
const config = {
  plugins: {
    '@tailwindcss/postcss': {},
  },
}

export default config
```

### `next.config.ts`

Minimal config — extend as needed:

```ts
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {}

export default nextConfig
```

### `tsconfig.json`

Standard Next.js TypeScript config with `@` alias:

```json
{
  "compilerOptions": {
    "target": "ES2017",
    "lib": ["dom", "dom.iterable", "esnext"],
    "allowJs": true,
    "skipLibCheck": true,
    "strict": true,
    "noEmit": true,
    "esModuleInterop": true,
    "module": "esnext",
    "moduleResolution": "bundler",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "jsx": "preserve",
    "incremental": true,
    "plugins": [{ "name": "next" }],
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  },
  "include": ["next-env.d.ts", "**/*.ts", "**/*.tsx", ".next/types/**/*.ts"],
  "exclude": ["node_modules"]
}
```

### `components.json`

Shadcn configuration. Generated by `pnpm dlx shadcn@latest init`. Note `"rsc": true` for React Server Components:

```json
{
  "$schema": "https://ui.shadcn.com/schema.json",
  "style": "default",
  "rsc": true,
  "tsx": true,
  "tailwind": {
    "config": "",
    "css": "src/app/globals.css",
    "baseColor": "neutral",
    "cssVariables": true
  },
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils",
    "ui": "@/components/ui",
    "lib": "@/lib",
    "hooks": "@/hooks"
  },
  "iconLibrary": "lucide"
}
```

### `src/app/layout.tsx`

Root layout — loads fonts, sets metadata, wraps all pages:

```tsx
import type { Metadata } from 'next'
import './globals.css'

export const metadata: Metadata = {
  title: 'PLUMBERITO',
}

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <head>
        <link
          href="https://fonts.googleapis.com/css2?family=Space+Grotesk:wght@300;400;500;600;700;800;900&family=JetBrains+Mono:wght@400;700&display=swap"
          rel="stylesheet"
        />
      </head>
      <body className="min-h-screen">
        {children}
      </body>
    </html>
  )
}
```

### `src/app/page.tsx`

Home route at `/` — a React Server Component by default:

### `src/lib/utils.ts`

Required by every Shadcn component:

```ts
import { clsx, type ClassValue } from 'clsx'
import { twMerge } from 'tailwind-merge'

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs))
}
```

## Scaffold steps (for an LLM agent)

These are the ordered steps to bring this template to a working state from scratch:

1. `pnpm create next-app . --typescript --tailwind --eslint --app --src-dir --import-alias "@/*" --empty` — initialize Next.js with App Router and TypeScript
2. Replace the auto-generated `tailwind.config.ts` and `postcss.config.mjs` with the v4 PostCSS config shown above — Tailwind v4 does not use a config file
3. Generate a `.gitignore` covering: `node_modules/`, `.next/`, `out/`, `dist/`, `*.tsbuildinfo`, `.env*`, `.pnpm-store/`, editor dirs, OS files, coverage output
4. Install Tailwind v4: `pnpm add -D tailwindcss @tailwindcss/postcss`
5. Replace `src/app/globals.css` with `@import "tailwindcss";` plus the `@theme` block and `body` noise texture
6. Update `src/app/layout.tsx` with the Google Fonts link tags and `metadata.title = 'PLUMBERITO'`
7. Replace `src/app/page.tsx` with the PLUMBERITO home component shown above
8. Install Shadcn minimum dependencies: `pnpm add class-variance-authority clsx tailwind-merge lucide-react @radix-ui/react-slot`
9. Create `src/lib/utils.ts` with the `cn()` helper
10. Create `components.json` with `"rsc": true` and point `tailwind.css` to `src/app/globals.css`
11. Optionally initialize Shadcn CLI: `pnpm dlx shadcn@latest init` (will detect existing `components.json`)
12. Run `pnpm dev --turbopack` to verify

## Adding pages

Create a `page.tsx` file inside a folder under `src/app/`. Next.js App Router uses the filesystem as the route tree.

```
src/app/about/page.tsx          →  /about
src/app/blog/page.tsx           →  /blog
src/app/blog/[slug]/page.tsx    →  /blog/:slug
```

Add a `layout.tsx` alongside any `page.tsx` to create a nested layout for that segment.

## Server vs Client components

All components in `src/app/` are **React Server Components** by default. Add `'use client'` at the top of a file to opt into client-side rendering:

```tsx
'use client'

import { useState } from 'react'
```

## Adding Shadcn components

```bash
pnpm dlx shadcn@latest add <component-name>
```

Components are added to `src/components/ui/` and can be imported as `@/components/ui/<component>`.
